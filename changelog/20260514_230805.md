# 20260514_230805

## Summary

合并 SP-1（sp1-spine，26 commits）到 main 后,merge 验证暴露了一个 fresh-checkout 构建问题:workspace 库包(`@cogni/contract`、`@cogni/shared`)的 `package.json` `main` 指向 gitignored 的 `dist/`,新克隆 + `pnpm install` 后未构建,导致任何 value-import 这两个包的测试 / typecheck / desktop build 全部失败。本次提交修复,使所有验证 gate 在干净检出上都能跑通。

## Changes

### 测试:vitest 从源码解析 workspace 包
- `vitest.config.ts` — 新增 `resolve.alias`,把 `@cogni/contract` / `@cogni/shared` 指到各自的 `src/index.ts`。vitest 本就即时转译 TS,这样测试无需任何构建步骤。

### 构建脚本:自给自足 + 修复 glob bug
- `package.json`(根)— `build` 脚本的 `--filter ./packages/*` glob 未加引号,会被 shell 提前展开,pnpm 把多余路径当成 script 名 → 一直是坏的(此前没人跑过根 `build`,各 task 都按包名单独 build)。加单引号修复。
- `package.json`(根)— `typecheck` 改为 `pnpm run build && pnpm -r typecheck`,先构建库包再 typecheck(cloud/runner-host 通过 project references 依赖库包的 `dist/.d.ts`)。
- `apps/desktop/package.json` — `build` / `typecheck` 前置 `pnpm --filter @cogni/contract build`,使 desktop 包能独立跑通。

### 文档
- `docs/RUNNING.md` §4 — 由「只 build runner-host」改为「`pnpm build` 构建全部库包」,并说明 `cloud dev`(tsx)/ desktop sidecar 都依赖 `dist/`。

- `pnpm build` → 4 库包全部 Done
- `pnpm typecheck` → 5 个 project(含 desktop)全部 Done
