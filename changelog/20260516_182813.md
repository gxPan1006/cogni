## Changes

### 桌面 ↔ 云端通信：补 CORS（不补就根本连不上）

- `packages/cloud/src/server.ts` — 在 `/api/*` 上挂 `hono/cors`，allow origin 限定 `tauri://localhost` 和 `http://localhost:1420`(vite dev)，并放行 `Authorization` header。
  - **直观表现**：之前桌面 webview 调 `localhost:8787/api/...` 会直接被浏览器 CORS 拦截，控制台一片红 + UI 永远是 loading 态。补完之后请求正常出去、回来。
  - **为什么之前没爆**：自动化测试用 fetch 直连 hono，绕过了浏览器 CORS 检查；只有跑真 Tauri webview 才会撞。

### 桌面端 deep-link：修「token 丢」+ 浏览器环境兼容

- `apps/desktop/src/useAuth.ts` —
  - 加 `isTauri()` 守卫：vite dev 在浏览器里跑也不再因为 `onOpenUrl` 不存在崩；浏览器环境的 `login()` 走 `window.location.href`。
  - 新增 `getCurrent()` 调用 + `acceptUrls` 公用 handler：app 启动时如果系统已经把 deep-link 投递过来但 `onOpenUrl` 还没注册，会丢 token；改成先 `getCurrent()` 把队列里已有的 URL 收掉，再挂监听。
  - `readToken()` 抽出来 + try/catch URL 解析。
  - **直观表现**：之前先在浏览器里完成 Google 流程、跳回 `cogni://auth?token=...`，app 偶尔会进 Login 页空等。补完之后 100% 进 chat shell。

### 桌面端 host 注册：修「云端有旧 host 行 + 本地 host.json 没了」的孤儿态

- `apps/desktop/src-tauri/src/daemon.rs` —
  - 新增 `has_host_config` 命令：检查 `~/.cogni/host.json` 是否存在。
  - `write_host_config` 在写新 config 前调用 `stop_recorded_daemon`：读 `~/.cogni/daemon.pid` → 给老 pid 发 SIGTERM → 删 pid 文件。Windows 端 stub。
- `apps/desktop/src-tauri/src/lib.rs` — 注册 `has_host_config` 命令。
- `apps/desktop/src/Shell.tsx` — 注册检查改成 `hosts.length === 0 || !hasHostConfig`：只要本地 host.json 没了就当 fresh 机器处理，重新 register + 写 config + 启 daemon。
  - **直观表现**：删了本地 `~/.cogni/`、再开 app → 之前会卡在「云端报你有 host 但本地没法认」的死循环；现在直接为本机重新注册一个 host、起 daemon、上线，黄色离线 banner 消失。

### SP-1 收尾：mint-dev-token 安全守卫

- `packages/cloud/src/scripts/mint-dev-token.ts`（新文件正式纳入仓库） —
  - 拒绝 `NODE_ENV=production`（防御性）。
